1. Conceptual Data Model

The conceptual model organizes the UBL analytics system around eight domains: league administration, competition structure, performance capture, physical environment, equipment, coaching, derived analytics, and system operations. Below are all core entities, cardinalities, and key constraints.

1.2 Critical Constraints

A Game belongs to exactly one Match and one Player (or one BakerSlot in baker format).

A Frame has exactly 2 rolls unless it is the 10th frame (up to 3 rolls) or the first ball is a strike (10th frame: 3 rolls).

Pins knocked per frame cannot exceed 10 unless a foul is recorded on roll 1 (frame resets to 10).

A Season must be closed (status = 'FINAL') before standings become immutable.

LaneSession dates must not overlap for the same lane_id.

Scorecard.verified_at locks that match against further sensor overwrites unless admin-flagged.

Example SQL: Player EWMA Form Index (last 10 games)

WITH ranked_games AS (

  SELECT player_id, scratch_score,

         ROW_NUMBER() OVER (PARTITION BY player_id

                             ORDER BY match_date DESC) AS rn

  FROM games g JOIN matches m USING (match_id)

  WHERE g.is_complete = TRUE

),

ewma_calc AS (

  SELECT player_id, scratch_score, rn,

         EXP(SUM(LN(0.7)) OVER (PARTITION BY player_id

             ORDER BY rn ROWS BETWEEN UNBOUNDED PRECEDING

             AND CURRENT ROW)) AS decay

  FROM ranked_games WHERE rn <= 10

)

SELECT player_id,

       SUM(scratch_score * decay) / SUM(decay) AS form_index_ewma

FROM ewma_calc

GROUP BY player_id;

5.2 Partitioning & Indexing

Partition match_rolls_fact and player_daily by season_id (range partition); sub-partition by month for large seasons.

Cluster/sort key order: player_id → team_id → match_date for player dashboards; reverse for team views.

Composite index on (player_id, season_id, match_date DESC) for trend queries.

GIN/JSONB index on match_events.metadata for event-type filtering.

Partial index: WHERE is_complete = TRUE on games table (excludes in-progress rows from stats queries).

6.2 Missing Data Handling

Missing frames: mark game is_complete=FALSE; exclude from averages until complete; flag in match_events as INCOMPLETE_GAME.

Sensor data absent: store manual pins_down only; sensor_quality=MISSING; all sensor-derived metrics become NULL — do not impute.

Makeup matches: status='MAKEUP_PENDING'; excluded from standings until verified_at is set.

Absent player/team (forfeit): insert game with scratch_score=0, add match_event FORFEIT; flag in scorecard.



6.3 Source-of-Truth Rules

PRIORITY ORDER: Verified Scorecard > Sensor Data > Manual Entry > Estimated/Imputed.

If scorecard.override_active=TRUE, ALL sensor data for that match is superseded.

Sensor overwrites manual only when sensor_quality=GOOD AND scorecard not yet verified.

Any conflict between sensor and manual that exceeds 5-pin tolerance triggers a CONFLICT event in ingestion_error_log and halts score finalisation.

9. Five Example Analytical SQL Queries



Query 1: Per-Player Recent Trend — Last 10 Games EWMA

-- EWMA form index for all active players, last 10 completed games

WITH game_ranked AS (

  SELECT g.player_id,

         g.scratch_score,

         m.match_date,

         ROW_NUMBER() OVER (PARTITION BY g.player_id

                             ORDER BY m.match_date DESC, g.game_number DESC) AS rn

  FROM games g

  JOIN matches m USING (match_id)

  WHERE g.is_complete = TRUE

),

weighted AS (

  SELECT player_id,

         scratch_score,

         rn,

         0.3 * POWER(0.7, rn - 1) AS weight   -- α=0.3

  FROM game_ranked WHERE rn <= 10

)

SELECT p.display_name,

       ROUND(SUM(w.scratch_score * w.weight) /

             SUM(w.weight), 2)       AS form_index_ewma,

       COUNT(*)                       AS games_in_window

FROM weighted w

JOIN players p USING (player_id)

GROUP BY p.display_name

ORDER BY form_index_ewma DESC;



Query 2: Team Standings — League Points + Tiebreakers

-- UBL standings: primary = total match points, then total pins

SELECT

  t.team_name,

  SUM(CASE WHEN m.home_team_id = t.team_id THEN m.home_points

           ELSE m.away_points END)           AS total_points,

  COUNT(CASE WHEN (m.home_team_id = t.team_id AND m.home_points >

                   m.away_points) OR

                  (m.away_team_id = t.team_id AND m.away_points >

                   m.home_points) THEN 1 END) AS wins,

  COUNT(CASE WHEN (m.home_team_id = t.team_id AND m.home_points <

                   m.away_points) OR

                  (m.away_team_id = t.team_id AND m.away_points <

                   m.home_points) THEN 1 END) AS losses,

  SUM(g.scratch_score)                        AS total_pins  -- tiebreaker

FROM teams t

JOIN matches m ON m.home_team_id = t.team_id

              OR m.away_team_id  = t.team_id

JOIN games g ON g.match_id = m.match_id AND g.team_id = t.team_id

WHERE m.status = 'COMPLETE'

  AND t.season_id = :season_id

GROUP BY t.team_id, t.team_name

ORDER BY total_points DESC, total_pins DESC;



Query 3: Clutch Performance — Last-Frame & Match-Deciding Games

-- Clutch index: avg score in match-deciding games vs overall avg

WITH player_overall AS (

  SELECT player_id,

         AVG(scratch_score) AS overall_avg

  FROM games WHERE is_complete = TRUE

  GROUP BY player_id

),

clutch_games AS (

  SELECT g.player_id,

         AVG(g.scratch_score) AS clutch_avg

  FROM games g

  JOIN matches m USING (match_id)

  WHERE g.is_complete = TRUE

  -- Deciding game = match won/lost by ≤ 1 point and this game was last

    AND g.game_number = (SELECT MAX(game_number) FROM games g2

                          WHERE g2.match_id = g.match_id

                            AND g2.player_id = g.player_id)

    AND ABS(m.home_points - m.away_points) <= 1

  GROUP BY g.player_id

)

SELECT p.display_name,

       ROUND(c.clutch_avg, 2)               AS clutch_avg,

       ROUND(o.overall_avg, 2)              AS overall_avg,

       ROUND(c.clutch_avg / o.overall_avg, 4) AS clutch_index

FROM clutch_games c

JOIN player_overall o USING (player_id)

JOIN players p USING (player_id)

ORDER BY clutch_index DESC;



Query 4: Split Conversion Leaders — Season

SELECT p.display_name,

       COUNT(*) FILTER (WHERE f.is_split)            AS splits_faced,

       COUNT(*) FILTER (WHERE f.split_converted)     AS splits_converted,

       ROUND(

         COUNT(*) FILTER (WHERE f.split_converted) * 100.0 /

         NULLIF(COUNT(*) FILTER (WHERE f.is_split), 0),

       1) AS split_pct

FROM frames f

JOIN games g USING (game_id)

JOIN matches m USING (match_id)

JOIN players p ON g.player_id = p.player_id

WHERE m.season_id = :season_id

  AND g.is_complete = TRUE

GROUP BY p.player_id, p.display_name

HAVING COUNT(*) FILTER (WHERE f.is_split) >= 5   -- min 5 splits for ranking

ORDER BY split_pct DESC, splits_converted DESC;



Query 5: Lane Bias Detection — Controlling for Player Ability

-- Identify lanes where players consistently over/underperform vs their average

WITH player_lane_avg AS (

  SELECT g.player_id,

         g.lane_id,

         AVG(g.scratch_score)   AS lane_avg,

         COUNT(*)                AS games_on_lane

  FROM games g

  WHERE g.is_complete = TRUE AND g.lane_id IS NOT NULL

  GROUP BY g.player_id, g.lane_id

  HAVING COUNT(*) >= 4

),

player_overall AS (

  SELECT player_id, AVG(scratch_score) AS overall_avg

  FROM games WHERE is_complete = TRUE

  GROUP BY player_id

)

SELECT

  l.lane_number,

  v.venue_name,

  ROUND(AVG(pla.lane_avg - po.overall_avg), 2) AS avg_lane_bias,

  COUNT(DISTINCT pla.player_id)                 AS distinct_players,

  SUM(pla.games_on_lane)                        AS total_games

FROM player_lane_avg pla

JOIN player_overall po USING (player_id)

JOIN lanes l ON pla.lane_id = l.lane_id

JOIN venues v ON l.venue_id = v.venue_id

GROUP BY l.lane_id, l.lane_number, v.venue_name

HAVING COUNT(DISTINCT pla.player_id) >= 3

ORDER BY ABS(AVG(pla.lane_avg - po.overall_avg)) DESC;





10. Dashboard & Visualization Recommendations



Dashboard 1: Player Profile Dashboard

Central view for individual player performance, trends and development.



Rolling Average Line Chart: last 20 games scratch score + 5-game EWMA overlay. X=game date, Y=score. Highlights slumps and peaks at a glance.

Strike / Spare / Open Breakdown Donut: season-level frame outcome distribution in three segments. Colour-coded green/amber/red. Drills to frame-level detail.

Per-Frame Score Heatmap: 10 columns (frames) × N rows (games). Cell colour = frame score percentile. Instantly reveals consistent weak frames (e.g., frame 9 spare shooting).



Dashboard 2: Team Standings Dashboard

League-wide competitive overview with rankings and head-to-head records.



Standings Table Widget: rank, team name, W-L, points, total pins, +/- vs prior week. Sortable. Click row to drill to team profile.

Team Trend Lines: multi-line chart — one line per team, weekly cumulative points. Reveals momentum shifts mid-season.

Head-to-Head Matrix: N×N heatmap grid of teams; cell = points won in head-to-head. Identifies persistent rivalries or mismatches.



Dashboard 3: Match Live Scorecard

Real-time match view during bowling sessions.



Live Rolling Scoresheet: per-player frame-by-frame grid updating via WebSocket. Strikes/spares shown as X/_ symbols.

Momentum Bar Chart: running pin differential between home and away team per game. Shifts from blue to red as lead changes.

Player vs Season Average Bar: for each active bowler, current game score vs their season average. Instantly shows who is bowling above/below form.



Dashboard 4: Coach Development Dashboard

Coachable insights for player development — requires sensor data and practice session access.



Ball Speed & Rev Rate Scatter: one dot per game; colour = scratch score decile. Reveals optimal release window and where deviations correlate with poor scoring.

Spare Type Conversion Funnel: by spare type (single pin, multi-pin, washout) — attempts vs conversions. Prioritises which spare drills are needed.

Frame Consistency Band Chart: mean ± 1 StdDev frame score per frame number. Wide bands flag inconsistency. Overlays practice vs match data.



Dashboard 5: Season Analytics Summary

End-of-season or mid-season executive overview for league administration.



Top 10 Averages Leaderboard: horizontal bar chart, men and women separate. Colour = most-improved indicator.

Strike Rate Distribution Histogram: all bowlers on x-axis buckets (0–10%, 10–20%, …, 60%+). Shows league talent distribution.

High Game & High Series Scatter: each bowler as a dot, HGS on x, HGS on y, dot size = games played. Outliers are peak performers.



Dashboard 6: Lane Intelligence Dashboard

Venue and lane-condition analytics for league managers and coaches.



Lane Bias Heatmap: lanes on X, player clusters on Y, cell colour = avg deviation from player baseline. Blue=lane plays easy, red=plays hard.

Oil Condition Degradation Line: average scratch score across games ordered by games_on_surface for the same pattern. Visualises carry-down effect.

Home vs Away Split Bar Chart: for each team, home and away average score side by side. Quantifies home advantage.
