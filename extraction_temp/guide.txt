1. Standings Projection & Title Race

With 4 weeks left in a 14-week season and full W/L data available for all teams in both divisions, you can project final standings, calculate points gaps, and identify which teams can still win, finish top 3, or are effectively eliminated. This is the single highest-engagement analysis for every league participant.



Why Start Here

You have 10 weeks of cumulative W/L data for 8 teams × 2 divisions. The weekly pairings for weeks 11 and 12 are explicitly in the PDFs. This gives you everything needed to model every possible finish scenario right now.

Core SQL Logic

-- Current standings with points gap to leader

WITH leader AS (

  SELECT division, MAX(total_points) AS top_points

  FROM season_standings GROUP BY division

)

SELECT s.team_name, s.division,

       s.total_wins, s.total_losses,

       s.total_points,

       s.total_points - l.top_points AS points_gap,

       s.total_pins,

       RANK() OVER (PARTITION BY s.division

                    ORDER BY s.total_points DESC,

                             s.total_pins DESC) AS current_rank

FROM season_standings s

JOIN leader l USING (division)

ORDER BY s.division, current_rank;



-- Max possible points remaining (4 weeks × 3 pts/match = 12 pts)

SELECT s.team_name,

       s.total_points AS current_pts,

       s.total_points + 12 AS max_possible_pts,

       l.top_points AS leader_pts,

       CASE WHEN s.total_points + 12 >= l.top_points

            THEN 'Still in title race'

            ELSE 'Eliminated from 1st'

       END AS title_status

FROM season_standings s

JOIN (SELECT division, MAX(total_points) top_points

      FROM season_standings GROUP BY division) l USING (division)

ORDER BY s.division, s.total_points DESC;



Monday Division Example (from data)

Playmasters lead at 26 pts with 12 pts remaining. 4Baggerz at 20 pts (6 behind) can reach max 32 — must win all 4 remaining AND Playmasters must lose all 4. Amigos Segundo at 24 pts (2 behind) can realistically catch up. Everyone else is 8+ behind — mathematically still possible but probability <5%.

2. Player Score Trend & Form Index

Every player in both divisions has cumulative season pins, games played, and season average — plus their week 9 and week 10 individual game scores. This lets you calculate improving/declining form, week-over-week movement, and rank players by current trajectory rather than just raw average.



Why Start Here

The Most Improved leaderboard in the PDFs already hints at this — DUFLA is +31.85 pins above their start-of-season average on Monday; WONI is +19.10 on Tuesday. But the raw list doesn't show trajectory direction, context, or whether players are still improving. You can build a full form dashboard from the data already in Supabase.

Master player list

score_id, player_id, week, game_number (1 or 2), scratch_score, match_id

Core SQL Logic

-- Week-over-week average movement

WITH weekly_avg AS (

  SELECT player_id,

         week,

         AVG(scratch_score) AS week_avg,

         COUNT(*)           AS games_that_week

  FROM game_scores

  GROUP BY player_id, week

),

with_lag AS (

  SELECT player_id, week, week_avg,

         LAG(week_avg) OVER (PARTITION BY player_id

                             ORDER BY week) AS prev_week_avg

  FROM weekly_avg

)

SELECT p.display_name,

       w.week,

       ROUND(w.week_avg, 1)                    AS this_week,

       ROUND(w.prev_week_avg, 1)               AS last_week,

       ROUND(w.week_avg - w.prev_week_avg, 1)  AS change

FROM with_lag w

JOIN players p USING (player_id)

ORDER BY ABS(w.week_avg - w.prev_week_avg) DESC;



-- EWMA form index (alpha=0.3, rolling all games)

WITH ordered AS (

  SELECT player_id, scratch_score,

         ROW_NUMBER() OVER (

           PARTITION BY player_id ORDER BY week, game_number) AS seq

  FROM game_scores

),

ewma AS (

  SELECT player_id, seq, scratch_score,

         AVG(scratch_score) OVER (

           PARTITION BY player_id

           ORDER BY seq

           ROWS BETWEEN 4 PRECEDING AND CURRENT ROW) AS rolling_5_avg

  FROM ordered

)

SELECT p.display_name, e.rolling_5_avg AS form_index

FROM ewma e

JOIN players p USING (player_id)

WHERE e.seq = (SELECT MAX(seq) FROM ewma e2

               WHERE e2.player_id = e.player_id)

ORDER BY form_index DESC;

3. Game 1 vs Game 2 Performance Split

Every UBL match consists of exactly two games. Every recap sheet in the dataset shows both game scores individually. This enables a unique analysis: identifying players and teams that consistently improve or fade across the two-game block — a directly coachable insight.



Why This Is Uniquely Valuable

Most bowling stats systems only show series totals. The UBL data captures game-level splits, which reveals patterns invisible in averages alone. A player averaging 175 could be consistently 165 / 185 (strong finisher) or 185 / 165 (fades under pressure). Teams exhibit the same patterns.



Supabase Table Structure

Uses the same game_scores table from Analysis 2. No additional tables required. The game_number column (1 or 2) is the key split dimension.



Core SQL Logic

-- Player game 1 vs game 2 average split

SELECT

  p.display_name,

  p.team_name,

  ROUND(AVG(CASE WHEN g.game_number = 1

                 THEN g.scratch_score END), 1) AS avg_game1,

  ROUND(AVG(CASE WHEN g.game_number = 2

                 THEN g.scratch_score END), 1) AS avg_game2,

  ROUND(AVG(CASE WHEN g.game_number = 2

                 THEN g.scratch_score END) -

        AVG(CASE WHEN g.game_number = 1

                 THEN g.scratch_score END), 1) AS g2_minus_g1,

  COUNT(DISTINCT g.week) AS weeks_bowled

FROM game_scores g

JOIN players p USING (player_id)

GROUP BY p.player_id, p.display_name, p.team_name

HAVING COUNT(DISTINCT g.week) >= 2

ORDER BY g2_minus_g1 DESC;



-- Team-level game 1 vs game 2 (sum of 4 bowlers)

SELECT

  t.team_name,

  SUM(CASE WHEN g.game_number = 1 THEN g.scratch_score END) AS team_g1_total,

  SUM(CASE WHEN g.game_number = 2 THEN g.scratch_score END) AS team_g2_total,

  SUM(CASE WHEN g.game_number = 2 THEN g.scratch_score END) -

  SUM(CASE WHEN g.game_number = 1 THEN g.scratch_score END) AS g2_advantage

FROM game_scores g

JOIN players p USING (player_id)

JOIN teams t ON p.team_id = t.team_id

GROUP BY t.team_id, t.team_name

ORDER BY g2_advantage DESC;



Real Example from Data

Monday Week 10: 4Baggerz bowled 769 in game 1 and 753 in game 2 (−16). Amigos Segundo bowled 741 game 1 and 571 game 2 (−170) — a dramatic collapse in game 2 despite having good individuals. This is exactly the kind of pattern this analysis surfaces.



Supabase Setup Steps

Confirm game_number (1 or 2) is populated for all game_scores rows. This maps directly from the recap sheet layout.

Create a VIEW player_game_split with avg_game1, avg_game2, split_delta, and consistency_label (Strong Finisher / Fader / Consistent).

Add a computed column: split_label = CASE WHEN g2_minus_g1 > 10 THEN 'Strong Finisher' WHEN g2_minus_g1 < -10 THEN 'Fader' ELSE 'Consistent' END.

4. Head-to-Head Matchup Breakdown

Every recap sheet gives the exact scores for each game in each matchup. This dataset contains 8 complete matchups across 2 weeks across both divisions — enough to build a full head-to-head matrix, margin-of-victory analysis, and lineup contribution tracking.



What This Unlocks

Beyond the W/L record, head-to-head data answers questions like: Which team wins when it's close? Which individual bowler consistently outperforms their counterpart in matchups? Which team is the most inconsistent in results? These drive engagement — every bowler wants to know who beats who.



Supabase Table Structure

Table

Key Columns

Notes

matches

match_id, week, division, home_team_id, away_team_id, home_points, away_points, home_series, away_series

One row per match

match_game_totals

match_id, game_number, home_score, away_score, winner (home/away/draw)

Two rows per match

player_match_scores

score_id, match_id, player_id, team_id, game_number, scratch_score

Individual bowler scores per match game



Core SQL Logic

-- Head-to-head matrix (points won between every pair)

SELECT

  t1.team_name AS team,

  t2.team_name AS opponent,

  SUM(CASE WHEN m.home_team_id = t1.team_id

           THEN m.home_points ELSE m.away_points END) AS pts_won,

  SUM(CASE WHEN m.home_team_id = t1.team_id

           THEN m.away_points ELSE m.home_points END) AS pts_conceded

FROM matches m

JOIN teams t1 ON t1.team_id IN (m.home_team_id, m.away_team_id)

JOIN teams t2 ON t2.team_id IN (m.home_team_id, m.away_team_id)

             AND t2.team_id != t1.team_id

GROUP BY t1.team_id, t1.team_name, t2.team_id, t2.team_name;



-- Margin of victory / defeat per match

SELECT

  ht.team_name AS home_team,

  at.team_name AS away_team,

  m.week,

  m.home_series, m.away_series,

  ABS(m.home_series - m.away_series) AS pin_margin,

  CASE WHEN m.home_series > m.away_series THEN ht.team_name

       WHEN m.away_series > m.home_series THEN at.team_name

       ELSE 'Draw' END                  AS series_winner,

  m.home_points, m.away_points

FROM matches m

JOIN teams ht ON m.home_team_id = ht.team_id

JOIN teams at ON m.away_team_id = at.team_id

ORDER BY m.week, pin_margin DESC;

Create a VIEW upset_detector: matches where the lower-ranked team won.

6. Participation Rate & Roster Attendance

The Bowler Statistics section flags which players bowled each week (LW column shows week number; 0 means absent). This lets you calculate attendance rates, identify teams relying on a small core, and flag availability risks.



Supabase Logic

-- Attendance rate per player

SELECT p.display_name, t.team_name,

       COUNT(CASE WHEN g.scratch_score > 0 THEN 1 END) AS weeks_bowled,

       s.total_weeks,

       ROUND(100.0 * COUNT(CASE WHEN g.scratch_score > 0 THEN 1 END)

             / s.total_weeks, 1)                        AS attendance_pct

FROM players p

JOIN game_scores g USING (player_id)

JOIN teams t USING (team_id)

JOIN seasons s ON s.season_id = p.season_id

GROUP BY p.player_id, p.display_name, t.team_name, s.total_weeks

ORDER BY attendance_pct DESC;



Key Setup Note

10. Team Consistency & Variance

Using team series totals across weeks, you can calculate which teams score consistently (low variance) vs which are unpredictable. Consistent teams are harder to upset; volatile teams may spike but also collapse — visible in the data.



Supabase Logic

-- Team consistency: StdDev of weekly series totals

SELECT t.team_name,

       ROUND(AVG(m.home_series + m.away_series) / 2, 0) AS avg_team_series,

       ROUND(STDDEV(

         CASE WHEN m.home_team_id = t.team_id THEN m.home_series

              ELSE m.away_series END), 0)                AS series_stddev,

       MIN(CASE WHEN m.home_team_id = t.team_id THEN m.home_series

                ELSE m.away_series END)                  AS worst_series,

       MAX(CASE WHEN m.home_team_id = t.team_id THEN m.home_series

                ELSE m.away_series END)                  AS best_series

FROM teams t

JOIN matches m ON t.team_id IN (m.home_team_id, m.away_team_id)

GROUP BY t.team_id, t.team_name

ORDER BY series_stddev ASC;

11. Lineup Contribution Analysis

Within each match, the 4-bowler lineup can be evaluated for individual contribution. Which bowler carries the team? Which position consistently underperforms? This is directly available from the player_match_scores table.



-- Bowler contribution as % of team total

SELECT p.display_name,

       SUM(g.scratch_score)                         AS total_pins,

       SUM(ts.team_total)                           AS team_total_over_matches,

       ROUND(100.0 * SUM(g.scratch_score) /

             SUM(ts.team_total), 1)                 AS contribution_pct

FROM game_scores g

JOIN players p USING (player_id)

JOIN (SELECT match_id, team_id, SUM(scratch_score) AS team_total

      FROM game_scores JOIN players USING (player_id)

      GROUP BY match_id, team_id) ts

  ON ts.match_id = g.match_id AND ts.team_id = p.team_id

GROUP BY p.player_id, p.display_name

ORDER BY contribution_pct DESC;

12. Remaining Schedule & Strength-of-Opponent

The PDFs include explicit pairings for weeks 11 and 12 in both divisions. Combined with current standings, you can calculate each team's remaining schedule difficulty and model expected points outcomes.



Supabase Logic

-- Remaining schedule difficulty (avg opponent rank)

WITH future_matches AS (

  SELECT sc.week, sc.home_team_id AS team_id,

         ss.rank AS opp_rank

  FROM schedule sc

  JOIN season_standings ss ON ss.team_id = sc.away_team_id

  WHERE sc.week > 10

  UNION ALL

  SELECT sc.week, sc.away_team_id,

         ss.rank AS opp_rank

  FROM schedule sc

  JOIN season_standings ss ON ss.team_id = sc.home_team_id

  WHERE sc.week > 10

)

SELECT t.team_name,

       ROUND(AVG(f.opp_rank), 1) AS avg_remaining_opp_rank,

       COUNT(*) AS remaining_matches

FROM future_matches f

JOIN teams t USING (team_id)

GROUP BY t.team_id, t.team_name

ORDER BY avg_remaining_opp_rank DESC;

14. Pins Over Average — Weekly Award Tracking

The 'Last Week Highs' section explicitly lists Pins Over Average winners (1st and 2nd) each week per division. On Monday Week 10, Parth led with +64. On Tuesday Week 9, Sharu led with +96. This is a weekly engagement metric — who bowled their best relative to expectations.



Supabase Logic

-- Pins over average per game

SELECT p.display_name,

       g.week,

       g.scratch_score,

       ps.season_avg,

       ROUND(g.scratch_score - ps.season_avg, 0) AS pins_over_avg

FROM game_scores g

JOIN players p USING (player_id)

JOIN player_season_stats ps USING (player_id)

ORDER BY pins_over_avg DESC;
